# .gitignore 研究文档

## 场景与职责

`.gitignore` 是 Git 版本控制系统的排除配置文件，定义了哪些文件/目录不应被 Git 跟踪和提交。在 Pretext 项目中，它负责维护仓库的整洁性，确保只有源代码和必要配置进入版本控制，而构建产物、依赖缓存、日志和本地环境文件被排除在外。

## 功能点目的

### 1. 依赖管理排除
- **`node_modules`**：排除 npm/bun 安装的依赖包目录
- 目的：依赖应通过 `bun.lock` 或 `package.json` 重建，而非提交到仓库

### 2. 构建输出排除
- **`out`**, **`dist`**, **`site`**：构建输出目录
- **`*.tgz`**：npm pack 生成的压缩包
- 目的：构建产物是派生文件，应在 CI/CD 或本地按需生成

### 3. 代码覆盖率排除
- **`coverage`**, **`*.lcov`**：测试覆盖率报告
- 目的：覆盖率数据是临时生成的分析结果

### 4. 日志文件排除
- **`logs`**, **`_.log`**：应用日志
- **`report.[0-9]_.[0-9]_.[0-9]_.[0-9]_.json`**：特定格式的报告文件
- 目的：日志是运行时产生的一次性数据

### 5. 环境变量排除
- **`.env`**, **`.env.development.local`**, **`.env.test.local`**, **`.env.production.local`**, **`.env.local`**
- 目的：环境变量包含敏感信息和本地配置，不应共享

### 6. 缓存排除
- **`.eslintcache`**：ESLint 缓存
- **`.cache`**：通用缓存目录
- **`*.tsbuildinfo`**：TypeScript 增量编译信息
- 目的：缓存文件可重建，且可能包含平台特定路径

### 7. IDE 和系统文件排除
- **`.idea`**：IntelliJ/WebStorm 配置
- **`.DS_Store`**：macOS Finder 元数据
- 目的：避免提交编辑器/操作系统特定的文件

## 具体技术实现

### 语法规则
- 每行一个模式
- `#` 开头为注释
- 空行被忽略
- 通配符：`*` 匹配任意字符（除 `/`），`?` 匹配单个字符
- 以 `/` 开头表示相对于仓库根目录
- 以 `/` 结尾表示目录

### 关键模式解析
```
report.[0-9]_.[0-9]_.[0-9]_.[0-9]_.json
```
- 使用字符类 `[0-9]` 匹配数字
- `_` 作为字面量匹配（可能是特定工具生成的报告格式）

## 关键代码路径与文件引用

### 相关文件
| 文件 | 关系 |
|------|------|
| `package.json` | 定义了 `prepack` 脚本生成 `dist/`，`site:build` 生成 `site/` |
| `tsconfig.build.json` | 配置 `outDir: "./dist"`，与 `.gitignore` 中的 `dist` 对应 |
| `bun.lock` | 依赖锁定文件（被 Git 跟踪），与 `node_modules` 排除形成对比 |

### 构建流程关联
```
prepack hook -> tsc -p tsconfig.build.json -> dist/ (被排除)
site:build -> scripts/build-demo-site.ts -> site/ (被排除)
bun install -> node_modules/ (被排除)
```

## 依赖与外部交互

### 与包管理器的交互
- **Bun**：项目使用 Bun 作为包管理器，`bun.lock` 替代 `package-lock.json`
- `node_modules` 排除是 Node.js/Bun 生态的标准做法

### 与 CI/CD 的交互
- `.github/workflows/pages.yml` 在 CI 中运行 `bun install --frozen-lockfile` 重建依赖
- 构建产物（`site/`）在 CI 中生成并部署，不存储在仓库中

### 与 TypeScript 的交互
- `*.tsbuildinfo` 排除配合 `tsconfig.json` 中的增量编译配置

## 风险、边界与改进建议

### 风险点
1. **dist/ 排除风险**：如果开发者忘记运行 `prepack`，可能缺少构建产物进行本地测试
2. **.env 排除风险**：新开发者可能不清楚需要创建 `.env` 文件，导致运行时错误
3. **缓存排除风险**：`.eslintcache` 被排除可能导致 CI 中 ESLint 运行变慢（但确保一致性）

### 边界情况
- `.gitignore` 不适用于已跟踪文件：如果文件已被 Git 跟踪，再加入 `.gitignore` 不会生效
- 子目录 `.gitignore`：当前项目根目录单一 `.gitignore` 足够，若未来有复杂子项目可能需要分层配置

### 改进建议
1. **添加文档注释**：在 `.gitignore` 顶部添加项目特定的注释说明
   ```gitignore
   # Pretext - Text layout library
   # See DEVELOPMENT.md for build process
   ```

2. **考虑添加**：
   ```gitignore
   # Editor configs
   .vscode/
   *.swp
   *.swo
   *~
   
   # OS files
   Thumbs.db
   ```

3. **验证脚本**：可添加一个 CI 检查确保没有意外提交被排除的文件类型

4. **与 package.json 同步**：当添加新的构建输出目录时，确保同时更新 `.gitignore` 和 `package.json` 的 `files` 字段
